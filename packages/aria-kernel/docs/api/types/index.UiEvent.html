    UiEvent — ui ↔ headless 통신의 단일 어휘. DOM Event global과 충돌 방지를 위해 Ui prefix 명시.

    Type Declaration

    • { dir?: NavigateDir; id?: string; type: "navigate" }

      navigate — id ⊕ dir (result-form ⊕ intent-form). schema.ts 의 zod refine 으로 런타임 강제.

    • { id: string; type: "activate" }
    • { id: string; open: boolean; type: "expand" }

      expand — inline content reveal (accordion section · disclosure twisty · tree branch). trigger 옆 콘텐츠가 in-flow 로 펼쳐짐. ARIA: aria-expanded on trigger. 대비: 'open' 은 popover/overlay (out-of-flow). 같은 aria-expanded 를 emit 해도 의도가 다름.

    • { anchor?: boolean; ids: string[]; to?: boolean; type: "select" }

      select — aria-selected 통합 어휘. 항상 ids 배열. to undefined ⇒ replace (ids 만 selected, 나머지 unselected). 단일 클릭 정본. to: true ⇒ additive set, to: false ⇒ unset. range/all/none batch. anchor: true ⇒ meta.selectAnchor 도 함께 갱신 (Space/Cmd+click 의 anchor reset). range select(Shift+Arrow) 는 anchor 유지해야 하므로 anchor 생략.

    • { ids: string[]; to?: boolean | "mixed"; type: "check" }

      check — aria-checked 통합 어휘. 항상 ids 배열. to undefined ⇒ 각 id 독립 토글. to: true|false|'mixed' ⇒ 일괄 set. 단일 토글은 ids: [id]. cascade/group 은 ids[].

    • { id: string; type: "value"; value: unknown }
    • { id: string; open: boolean; type: "open" }

      open — popover/overlay 가시성 (combobox listbox · menu · dialog · tooltip). out-of-flow overlay 가 떠오름/사라짐. ARIA: combobox aria-expanded, dialog aria-modal, 또는 conditional mount. 대비: 'expand' 는 inline reveal.

    • { buf: string; deadline: number; type: "typeahead" }
    • { dx: number; dy: number; id: string; type: "pan" }

      pan: target entity의 (x, y)를 (dx, dy)만큼 이동 — gesture 어댑터가 wheel/pointer를 번역

    • { cx: number; cy: number; id: string; k: number; type: "zoom" }

      zoom: cursor (cx, cy)를 고정점으로 scale을 k 배 — Figma/Miro 식 cursor-anchored zoom

    • { siblingId: string; type: "insertAfter"; value?: unknown }

      Edit / Clipboard / History 어휘 — 정본 = zod-crud JsonCrud op 1:1. 의도적으로 시그니처가 zod-crud 와 동일 (insertAfter / appendChild / update / delete / copy / cut / paste / undo / redo). 추상화 0, 옵션 0 — opinionated.

    • {
          id: string;
          mode: "child" | "sibling-after" | "sibling-before";

          type: "move";

      move — id 를 targetId 로 이동. clipboard 를 거치지 않음 (Tab demote/promote 같은 구조 편집은 사용자의 cut/copy buffer 를 오염시키면 안 됨). adapter 가 read + insert + delete 시퀀스로 구현하거나 backend native move 위임.

    • { type: "undo" }
    • { type: "redo" }
    • { id: string; type: "editStart" }

      editStart — 인라인 편집 진입(meta.editing = id). UI 가 input 으로 swap.

    • { type: "editEnd" }

      editEnd — 인라인 편집 종료(meta.editing = null). commit 은 별도 update 이벤트로.

    • { type: "selectAll" }

      selectAll — Cmd/Ctrl+A. 가시 항목 전체 선택. host reducer 가 select{ids,to:true} 로 풀거나 직접 처리.

    • { type: "selectNone" }

      selectNone — Esc. 현재 선택 비우기.

    • { to: string; type: "selectRange" }

      selectRange — Shift+Click/Arrow. anchor 부터 to 까지 범위 선택. anchor 는 meta.selectAnchor 사용.

    • { id: string; type: "focus" }

      focus — 프로그래매틱 포커스 설정. navigate { id } 와 다른 점: navigate 는 사용자 제스처라 sff 같은 gesture 가 select 부수효과를 합성하지만, focus 는 autoFocus·dialog 복귀·find 결과 점프처럼 부수효과 없는 순수 focus 이동.

    • { key: string; order: "asc" | "desc" | "none"; type: "sort" }

      sort — aria-sort 정본. key + order.

    • { query: string; type: "filter" }

      filter — collection query 입력 (combobox·grid·listbox). typeahead 와 다른 축.

    • { query?: string; type: "find" }

      find — Cmd/Ctrl+F. find UI 토글 또는 query 갱신.

    • { type: "save" }

      save — Cmd/Ctrl+S. host 가 persist.

    • { type: "commit" }

      commit — Enter on edit. 현재 편집을 확정. update 이벤트와 함께 dispatch 하기도 함.

    • { type: "revert" }

      revert — Esc on edit. 편집 폐기. editEnd 와 함께 dispatch.

    • { id: string; type: "duplicate" }

      duplicate — Cmd/Ctrl+D. id 를 형제로 복제. host 가 read+insert 로 풀음.

    • { parentId?: string; type: "new" }

      new — Cmd/Ctrl+N. 새 항목/문서 생성. parentId 생략 시 root.

    • { id?: string; type: "close" }

      close — Cmd/Ctrl+W. 윈도우·탭·다이얼로그 닫기. open{open:false} 와 다름 — 사용자 의도 dismiss.

    • { id?: string; type: "cancel" }

      cancel — Esc on dialog/popover. DOM <dialog> cancel event 정본. close 와 다름 — 확정 vs 취소.

    • { id?: string; type: "refresh" }

      refresh — Cmd/Ctrl+R / F5. data refetch 의도. id 생략 시 전체.

    • { type: "print" }

      print — Cmd/Ctrl+P.

    • { type: "goBack" }

      goBack — Cmd+[ / Backspace. 위치 히스토리 뒤로. undo(내용 히스토리)와 분리.

    • { type: "goForward" }

      goForward — Cmd+]. 위치 히스토리 앞으로.

    • { id?: string; type: "expandAll" }

      expandAll — Cmd+Opt+→. 트리 전체 펼침. id 생략 시 root 부터.

    • { id?: string; type: "collapseAll" }

      collapseAll — Cmd+Opt+←. 트리 전체 접음.

    • { query: string; type: "replace"; with: string }

      replace — Cmd+H. find & replace.

    • { type: "nextMatch" }

      nextMatch — F3. find 결과 다음.

    • { type: "prevMatch" }

      prevMatch — Shift+F3. find 결과 이전.

    • { id: string; type: "dragStart" }

      dragStart — DnD gesture 시작. HTML5 dragstart 정본.

    • { id: string; targetId: string; type: "dragOver" }

      dragOver — DnD 중 hover target 후보.

    • {
          id: string;
          mode?: "child" | "sibling-after" | "sibling-before";
          targetId: string;
          type: "drop";
      }

      drop — DnD 확정. move 와 다름 — drop 은 gesture intent, move 는 결과 op.

    • { id: string; type: "dragEnd" }

      dragEnd — DnD 종료(취소 포함).
