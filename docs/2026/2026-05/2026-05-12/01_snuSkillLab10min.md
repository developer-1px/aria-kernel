# 스킬 만들기 — 10분 실습 (서울대 4학년 비전공자)

## 한 줄 메시지

> **"스킬을 만든다 = AI가 언제 발동할지를 정확히 전달하고, 진짜 그렇게 발동하는지 시험한다"**

학생은 description을 손으로 쓰지 않는다. `skill-creator`가 써준다.
학생의 진짜 일은 두 가지:

1. **내 의도를 skill-creator에게 정확히 전달하기**
2. **나온 스킬이 진짜 작동하는지 시험하기 (3-shot 트리거 시험)**

10분 실습은 이 둘만 몸으로 익히게 한다.

---

## 타임라인 (10분)

### 0:00–2:00 · 데모 (교수)

같은 의도를 `skill-creator`에게 **다르게 말했을 때** 어떻게 다른 스킬이 나오는지 비교.

```
A) "코드 리뷰 스킬 만들어줘"
   → description이 광범위해짐 → 무관한 질문에도 발동

B) "git diff를 붙여넣고 피드백 요청할 때만 발동하는 코드 리뷰 스킬.
    일반 코딩 질문에는 발동 ❌"
   → description이 좁아짐 → 의도대로 발동
```

같은 프롬프트를 두 스킬에 던져 **트리거 차이**를 눈으로 보여준다.

> "차이는 여러분이 skill-creator에게 한 첫 마디입니다."

---

### 2:00–5:00 · 스킬 생성 (학생, 3분)

학생 손에 쥐어줄 카드:

```
─────────────────────────────────────────────
당신이 매주 반복하는 작업 1개를 떠올리세요.
(예: 영수증 정리 / 과제 요약 / 일정 메일 / 인용 형식 통일)

claude.ai에서 /skill-creator 호출.
첫 마디는 아래 3칸을 채워서 말하세요.

  "_______________ 할 때 발동하는 스킬을 만들고 싶어.
   _______________ 를 해주면 돼.
   단, _______________ 같은 비슷한 상황엔 발동하면 안 돼."

skill-creator가 묻는 질문에 대답해서 SKILL.md를 받으세요.
─────────────────────────────────────────────
```

핵심 가이드 (슬라이드 1장):

- **언제(When)** — 어떤 말·행동을 신호로 삼는가?
- **무엇(does)** — 그래서 무엇을 해주는가? (1개만)
- **NOT** — 비슷한데 발동하면 안 되는 경우 (가장 중요)

---

### 5:00–8:00 · 3-shot 트리거 시험 (학생, 3분)

받은 스킬에 시험 프롬프트 3개를 작성한다.

```
─────────────────────────────────────────────
✅ HIT  (반드시 발동해야 함):
   "_____________________________________"

❌ MISS-1 (비슷하지만 발동 ❌):
   "_____________________________________"

❌ MISS-2 (전혀 무관, 발동 ❌):
   "_____________________________________"
─────────────────────────────────────────────
```

3개를 차례로 claude.ai에 던진다.

- **3/3 통과** → ✅ Pass
- **1개라도 틀림** → 어느 칸이 틀렸는지 확인 → skill-creator에게 "이런 경우에 발동/안 발동하게 description 고쳐줘" 요청 → 재시험

이게 핵심 메커니즘이다:
- HIT 통과 → 의도가 description에 잘 들어갔다
- MISS-1 실패 → description이 너무 넓다 (false positive)
- MISS-2 실패 → description이 형편없다 (아무 데나 발동)

---

### 8:00–10:00 · 시연 + 마무리

자원자 1명이 **실패한 케이스**를 시연:
- 어떤 MISS가 잘못 발동했는지 보여주기
- skill-creator로 한 줄 수정 → 재시험 통과

마무리 한 줄:

> "스킬 작성의 본질은 코드도, 도구도 아닙니다.
> **내 의도를 정확히 전달하고, 진짜 그대로 작동하는지 시험하는 것**입니다.
> 여러분이 방금 한 게 그 전부예요."

---

## 산출물 (제출/평가 단위)

학생 1명당 한 쌍:

1. `skill-creator`로 생성된 SKILL.md
2. 3-shot 시험 프롬프트 + 통과 결과 스크린샷

**Pass 조건: 3/3 트리거 시험 통과.** 학점 아님, 자기 검증.

## 자주 막히는 지점 (TA 대비)

| 증상 | 처방 |
|------|------|
| "뭘 만들지 모르겠어요" | "지난주에 두 번 이상 한 일" 1개만 떠올리게 |
| 의도 첫 마디가 추상적 | When/does/NOT 3칸을 비워두지 못하게 강제 |
| MISS 칸을 못 채움 | "비슷한데 발동되면 짜증날 상황"으로 환원 |
| HIT은 통과, MISS 실패 | description이 너무 넓음 → "이런 경우엔 발동 ❌" 명시 요청 |
| 3개 다 실패 | 의도 첫 마디로 돌아가서 다시 말하기 |

## 왜 짝 검증이 아니라 3-shot 시험인가

| 짝 검증 | 3-shot 트리거 시험 |
|---|---|
| 사람이 판정 → 봐줌 가능 | LLM이 판정 → 정직 |
| description만 봄 | 스킬 전체 봄 |
| skill-creator 안 쓰는 척 | skill-creator 산출물 그대로 시험 |
| 양성 1개만 | 양성 1 + 음성 2 (false positive 잡힘) |

특히 **MISS 칸 2개**가 핵심. skill-creator가 광범위하게 써버리는 실수를 학생이 직접 잡는다.
